# CS-255
For this project, I worked with the DriverPass case study to design a stystem that would help students prepare for their DMV driving tests. DriverPass wanted a web-based system where customers could create an account, buy training packages, schedule driving lessons, take practice tests, and keep track of their progress. The system also needed features for employees like instructors, secretaries, administrators, and IT staff to manage customer information and user accounts. 
I think one of the things I did best was organizing the client's requirements. I made sure to include both the functional and nonfunctional requirements, along with security, browser compatibility, cloud storage, and different user roles. I also tried to think about how each type of user would interact with the system and what they would need to do. If I could go back and improve one part of my work, I would make the functional requirements more detailed and include more diagrams to add to showing how the system works. I think adding more visuals would make it easier to view how all the parts connect. 
I used the interview with the client to figure out what they needed and turn those needs into system requirements. I included features like secure log ins, online scheduling, practice tests, and progress tracking because those were all important to DriverPass. It's important to think about the user's needs because if the system isn't easy to use, it won't be successful regardless of how well it's built.
When thinking about software design, I like to start by understanding exactly what the client wants before thinking about how to build anything. From there, I organize the requirements and make sure every feature has a purpose. In the future, I want to continue using diagrams, planning out requirements first, and utilizing feedback. This project helped me understand how important it is to plan everything out before any coding is done.
